# IPC bindings

`bindings.ts` is auto-generated by `tauri-specta`. Do not edit it manually. Regenerate with:

```
cd apps/desktop/src-tauri && cargo nextest run ipc::tests::export_bindings_test
```

## Usage

```typescript
import { commands } from '$lib/ipc'

const result = await commands.greet('world')
```

Commands that aren't in `bindings.ts` yet (see "excluded commands" below) still use the old `invoke('...')` pattern via
the wrappers in `src/lib/tauri-commands/`.

## Architecture

The Rust side lives in `apps/desktop/src-tauri/src/ipc.rs`. Commands are registered in two parallel lists:

- `tauri::generate_handler![]` — runtime dispatch for all commands
- `specta::function::collect_functions![]` (per platform group) — type info for `bindings.ts` generation

Commands that can't be included in `collect_functions![]` (see below) still work at runtime via `generate_handler![]`.

## Excluded commands (specta rc.24 limitation)

specta-serde rc.24 splits types with `#[serde(skip_serializing_if = ...)]` fields into `_Serialize`/`_Deserialize` pairs
in `apply_phases()`. Then `validate_exported_command` (which uses `ApplyMode::Unified`) rejects the split types.
Removing `Deserialize` from the Rust derive does **not** fix this — the split is based on specta's stored
serde-attribute metadata, not on whether `Deserialize` is implemented.

As a result, commands that return or take types with `skip_serializing_if` fields are excluded from
`collect_functions![]` (they stay in `generate_handler![]` so runtime IPC works). They're tracked with comments in
`ipc.rs`.

Current exclusions:

| Command                                                 | Reason                                                                                                                                   |
| ------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `get_file_range`, `get_file_at`, `get_files_at_indices` | Return `Vec<FileEntry>` / `Option<FileEntry>` — `FileEntry` has optional fields                                                          |
| `update_left_pane_state`, `update_right_pane_state`     | Take `PaneState` which has optional fields                                                                                               |
| `list_mtp_devices`                                      | Returns `Vec<MtpDeviceInfo>` — has optional fields                                                                                       |
| `connect_mtp_device`, `get_mtp_device_info`             | Return `ConnectedDeviceInfo` containing `MtpDeviceInfo` + `Vec<MtpStorageInfo>`                                                          |
| `get_mtp_storages`                                      | Returns `Vec<MtpStorageInfo>` — has optional fields                                                                                      |
| `list_mtp_directory`                                    | Returns `Vec<FileEntry>`                                                                                                                 |
| `list_volumes`, `resolve_path_volume`                   | Return `Vec<LocationInfo>` / `PathVolumeResolution<LocationInfo>` — has optional fields                                                  |
| `list_network_hosts`, `resolve_host`                    | Return `Vec<NetworkHost>` / `Option<NetworkHost>` — has optional fields                                                                  |
| `connect_to_server`                                     | Returns `ManualConnectResult` containing `NetworkHost`                                                                                   |
| `prepare_error_report_preview`                          | Returns `PreviewPayload` containing `BundleManifest` — has optional fields                                                               |
| `check_pending_crash_report`, `send_crash_report`       | Use `serde_json::Value` — specta rc.24 inlines the recursive enum but emits `Vec<Value>` / `Map<string, Value>` as Rust type names in TS |
| `record_breadcrumb`, `record_settings_defaults`         | Take `Option<serde_json::Value>` / `HashMap<String, serde_json::Value>` — same issue                                                     |

## Decision: Option B (exclude from collect_functions, keep in generate_handler)

Why: The alternative (Option A) would be to remove `skip_serializing_if` from all affected types, changing the JSON wire
format from field-absent to `null`. That would require 174+ frontend changes and break patterns like
`if (recursiveSize !== undefined)`. Option B preserves the wire format and runtime functionality with minimal churn, at
the cost of those commands not having typed bindings until specta is upgraded.
